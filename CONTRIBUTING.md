# Contributing to GoGate

Thanks for contributing.

## Before You Start

- Read [README.md](./README.md) for local setup.
- Read [CLA.md](./CLA.md). Contributions are accepted only under the CLA terms.
- Keep pull requests focused and small.

## Development Workflow

1. Fork and create a feature branch.
2. Make changes with tests.
3. Run checks locally:

```bash
make lint
make test
make vet
```

4. Open a pull request with:
- clear summary
- linked issue (if any)
- test notes

## Coding Standards

- Follow idiomatic Go and keep functions focused.
- Add or update tests for changed behavior.
- Keep public behavior and docs in sync.

## Licensing of Contributions

By contributing, you agree that your contributions are provided under:

- the project's open-source license (AGPLv3), and
- additional rights described in [CLA.md](./CLA.md), including relicensing rights for dual-licensing.
